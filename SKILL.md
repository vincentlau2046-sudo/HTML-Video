## 技能定位
HTML-Video 是一个专业级技能，用于将技术资料（Markdown/文本）和HTML幻灯片转换为高质量的汇报视频。该技能专注于内容准确性、专业表达和视听质量，适用于技术分析、架构拆解、商业汇报等场景。

## 核心能力
- **内容理解**: 深度解析技术资料，提取核心要点和关键数据
- **讲稿生成**: 基于原始资料生成口语化、专业化的讲解脚本
- **音频合成**: 使用Edge TTS生成高质量语音，支持多语言和语速调节
- **视觉处理**: 正确处理HTML幻灯片，确保每页内容准确显示
- **视频合成**: 将音频和视觉元素合成为流畅的专业汇报视频

## 输入要求

### 必需输入
| 输入类型 | 格式要求 | 说明 |
|---------|---------|------|
| **HTML幻灯片** | `.html` 文件 | 必须包含完整的CSS样式和独立的页面结构 |
| **技术资料** | `.md` 或 `.txt` 文件 | 包含详细的技术分析、数据和核心发现 |
| **输出格式** | `video` | 固定值，表示生成汇报视频 |

### 可选参数
| 参数 | 默认值 | 选项 | 说明 |
|------|--------|------|------|
| **分辨率** | `1080p` | `720p`/`1080p`/`4k` | 视频输出分辨率 |
| **TTS引擎** | `edge-tts` | `edge-tts`/`stepfun`/`dashscope` | 语音合成引擎 |
| **音色** | `zh-CN-XiaoxiaoNeural` | [Edge TTS支持的音色列表](https://learn.microsoft.com/zh-cn/azure/ai-services/speech-service/language-support?tabs=tts) | 语音角色选择 |
| **语速** | `+25%` | `-20%` to `+50%` | 语速调节，推荐+25%提升信息密度 |
| **输出目录** | `./output/{project_name}` | 自定义路径 | 输出文件保存位置 |

## 处理流程

### 阶段1: 输入验证与预处理
1. **文件验证**: 确认HTML和Markdown文件存在且可读
2. **内容分析**: 解析HTML结构，识别独立页面数量和内容
3. **技术资料解析**: 提取核心要点、数据表格和关键发现
4. **兼容性检查**: 验证HTML是否包含必要的CSS样式和结构

### 阶段2: 讲稿生成
1. **开篇设计**: 创建简洁有力的开场白，介绍主题重要性
2. **逐页讲稿**: 为每页HTML内容生成对应的口语化讲解
3. **过渡衔接**: 添加自然的页间过渡词（"接下来我们看..."、"更重要的是..."）
4. **理解性举例**: 为复杂概念提供具体实例和生活化解释
5. **结尾总结**: 生成专业的结论和建议

### 阶段3: 音频生成
1. **TTS配置**: 设置指定的音色、语速和音频参数
2. **分页音频**: 为每页讲稿生成独立的音频文件
3. **质量验证**: 检查音频清晰度、时长和完整性
4. **格式标准化**: 确保音频格式兼容视频合成

### 阶段4: 视觉处理
1. **页面提取**: 从HTML中提取每页独立内容
2. **单页HTML生成**: 为每页创建包含完整样式的独立HTML文件
3. **截图生成**: 使用wkhtmltoimage将单页HTML转换为PNG图片
4. **质量验证**: 确保每张截图显示正确的内容（非重复页面）

### 阶段5: 视频合成
1. **单页视频**: 将每页截图和对应音频合成单页视频
2. **视频合并**: 按顺序合并所有单页视频为完整汇报视频
3. **质量检查**: 验证视频流畅性、音频同步和内容准确性
4. **输出整理**: 生成完整的输出目录结构和说明文件

## 输出规范

### 输出目录结构
```
{output_dir}/
├── {project_name}.md              # 原始技术资料备份
├── README.md                      # 输出说明文件
├── html/                          # HTML幻灯片源文件（如果提供）
│   └── index.html
└── video/
    ├── {project_name}_Complete.mp4     # 完整汇报视频
    ├── slices/                    # 分页讲稿Markdown文件
    │   ├── slide_01_intro.md
    │   ├── slide_02_content.md
    │   └── ...
    ├── screenshots/               # 幻灯片截图PNG文件
    │   ├── slide_01.png
    │   ├── slide_02.png
    │   └── ...
    ├── audio/                     # 分页音频MP3文件
    │   ├── slide_01.mp3
    │   ├── slide_02.mp3
    │   └── ...
    └── temp_videos/               # 单页视频临时文件
        ├── slide_01.mp4
        ├── slide_02.mp4
        └── ...
```

### 内容质量标准
- **专业性**: 符合技术汇报的专业标准
- **准确性**: 内容与原始资料完全一致
- **流畅性**: 语音自然，无卡顿或异常停顿
- **同步性**: 音频与对应幻灯片内容精确匹配
- **完整性**: 包含所有页面，无遗漏或重复

## 禁止/不及格事项（❌ 绝对禁止）

### 内容相关
1. **❌ 重复页面**: 所有幻灯片截图显示相同内容（这是严重错误）
2. **❌ 内容不匹配**: 音频讲解与幻灯片内容不符
3. **❌ 技术错误**: 歪曲或错误解释原始技术资料中的概念
4. **❌ 遗漏关键信息**: 跳过重要的数据点、图表或核心发现
5. **❌ 过度简化**: 将复杂技术概念过度简化导致失真

### 技术实现
6. **❌ 音频质量问题**: 语音不清晰、有杂音、语速异常
7. **❌ 视频质量问题**: 分辨率不足、画面模糊、色彩失真
8. **❌ 同步问题**: 音频与视频不同步，延迟超过1秒
9. **❌ 格式错误**: 输出文件无法正常播放或格式不标准
10. **❌ 编码问题**: 使用不兼容的编码格式导致播放失败

### 流程规范
11. **❌ 跳过验证步骤**: 未进行内容验证直接生成最终视频
12. **❌ 未经测试的依赖**: 使用未验证的工具或库导致失败
13. **❌ 不完整的输出**: 缺少必要的输出文件或目录结构
14. **❌ 无错误处理**: 遇到问题时不降级处理而直接失败
15. **❌ 未记录过程**: 不生成README说明文件和过程日志

### 质量控制
16. **❌ 未验证HTML结构**: 直接处理未验证的HTML文件
17. **❌ 未检查音频时长**: 音频时长与预期严重不符
18. **❌ 未测试播放兼容性**: 未在标准播放器中测试输出文件
19. **❌ 未备份原始文件**: 未保存原始输入文件的备份
20. **❌ 未提供回退方案**: 当主要流程失败时无备选方案

## 错误处理与降级策略

### 可恢复错误
| 错误类型 | 降级方案 | 用户通知 |
|----------|----------|----------|
| **TTS API失败** | 切换到备用TTS引擎或使用本地TTS | 告知用户语音质量可能受影响 |
| **截图工具失败** | 使用浏览器自动化替代wkhtmltoimage | 告知用户处理时间可能延长 |
| **部分页面失败** | 跳过失败页面，继续处理其他页面 | 明确指出哪些页面缺失 |
| **音频生成超时** | 降低音频质量参数重试 | 告知用户音频质量调整 |

### 不可恢复错误
| 错误类型 | 处理方案 |
|----------|----------|
| **输入文件损坏** | 立即终止，返回具体错误信息 |
| **HTML结构无效** | 终止处理，建议用户修复HTML结构 |
| **磁盘空间不足** | 清理临时文件后终止，提示空间需求 |
| **权限不足** | 终止处理，提示需要相应文件系统权限 |

## 性能优化建议

### 资源管理
- **内存优化**: 处理大文件时分块处理，避免内存溢出
- **CPU利用**: 并行处理独立任务（如多页音频生成）
- **磁盘IO**: 批量写入减少磁盘操作次数
- **缓存策略**: 重用已生成的中间文件避免重复计算

### 时间优化
- **预估时间**: 根据输入大小预估处理时间并告知用户
- **进度报告**: 定期报告处理进度，避免用户等待焦虑
- **增量处理**: 支持只重新处理修改过的页面
- **后台处理**: 支持长时间任务的后台执行

## 使用示例

### 基本用法
```bash
html-video --html-file "presentation.html" \
           --content-file "technical_analysis.md" \
           --output-dir "./output/my_presentation"
```

### 高级配置
```bash
html-video --html-file "architecture.html" \
           --content-file "deep_dive_analysis.md" \
           --resolution "1080p" \
           --voice "zh-CN-YunxiNeural" \
           --speed "+25%" \
           --output-dir "./output/architecture_presentation"
```

### 批量处理
```bash
# 处理多个项目
for project in project1 project2 project3; do
    html-video --html-file "${project}/slides.html" \
               --content-file "${project}/analysis.md" \
               --output-dir "./output/${project}"
done
```

## 注意事项
1. **HTML结构要求**: HTML必须包含明确的页面分隔（如独立的div元素）
2. **CSS兼容性**: 复杂的CSS动画可能无法正确渲染
3. **字体支持**: 确保系统安装了HTML中使用的字体
4. **网络依赖**: TTS生成需要网络连接
5. **处理时间**: 大型项目可能需要较长时间处理
